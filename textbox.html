Accessibility

Tab to focus in and focus out the element. Read error message if any as soon as the element gets off focused. For default behaviour of screen reader to work, use semantic HTML or alternatively, https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/textbox_role

On focus, as there is nothing hugely flagged on NVDA screen reader while entering values in auto format fields apart from reading back the commas, spaces and hyphen back to screen reader users.

NVDA does not read the auto format slashes but reads when user types in. Chromevox reads out the auto format slashes but not the spaces. Format field example: https://codepen.io/tutsplus/pen/KMWqRr

Inform that the particular format characters read out to screen reader users while entering values in the field before they land in the input field might be a better solution than format happen when user off focus from the input field. Cursor land on previous character or digit from the user's target when revisits to edit the input value. https://www.filamentgroup.com/lab/politespace.html

Date input

Auto tabbing is not good for screen readers as they will not know which field they are on

Option fields

By using optional users can figure out by default the rest is mandatory, giving a cleaner customer experience.

Often designers feel that the having a marker for every single required field is repetitive, ugly, takes too much space, and, with longer forms, may even seem oppressive (the form requires so much from the user!).

Disabled fields

Avoid using disabled textbox as screen readers will not detect them.
