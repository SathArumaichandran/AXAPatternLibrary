Table

Use the table component to make information easier to compare and scan for users.

Given the data variations and different presentation purposes, we recommend a customised design for the table. Please only refer to the below sections for the styling and best practice for usability and accessibility.

  • Avoiding tables with multiple levels of row or column headers and headers that span multiple rows or columns. Such tables may be difficult to implement with the correct markup and scope attributes
  • When comparing columns of numbers, align the numbers to the right in table cells for easy scanning

Accessibility

A critical step toward creating an accessible data table is to designate row and/or column headers. In the markup, the element is used for table data cells and the element is used for table header cells.

Avoiding blank table cells, especially when those table cells are used for formatting as it may not work for blind users. One alternative to using white space for formatting a single table may include using multiple tables.
